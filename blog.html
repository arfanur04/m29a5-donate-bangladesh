a. What is the Document Object Model (DOM)?

The Document Object Model (DOM) is like a blueprint or map of a webpage. When your browser loads an HTML file, it creates the DOM—a tree-like structure where every element (like paragraphs, images, or buttons) is a node.

b. How do you select an element from the DOM?

Well, there are a few ways to grab an element from the DOM (Document Object Model), depending on what you know about the element. It's like searching for something in a messy room – you can use different strategies.

some way :
- getElementById()
- querySelector()
- querySelectorAll()

c. What is event delegation in the context of the DOM, and why is it useful?

Event delegation is a technique where you add one event listener to a parent element to handle events for its child elements instead of adding individual listeners to each child. When an event (like a click) happens on a child element, it "bubbles up" to its parent, and the single listener can check which child triggered the event.

d. How do you manipulate an element's attributes and styles using the DOM?

You can change an element's attributes and styles directly using the DOM API. Here are a few ways to do this

- setAttribute()
- getAttribute()
- removeAttribute()
